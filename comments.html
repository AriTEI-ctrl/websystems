On this page we will explore the design principles, technical aspects, aesthetics and accessibility practices implemented while coding this website based on the guidelines and principles outlined by w3.org.

Design

Considering the design of my website, I endeavoured to achieve a minimalistic look while withholding adequate sophistication. The minimalistic design philosophy emphasizes clarity, elegance, and functionality which I adhered to via:

  • Clean Layout: An uncluttered layout with ample white space, allowing the content to breathe and create a sense of spaciousness.

  • Limited Colour Palette: Neutral tones of white, black, gray, and subtle shades of ocean green, to create a serene atmosphere.

  • Simplified Navigation: Streamlined navigation menu that guides you effortlessly to the information you need.

  • Responsive Design: A seamless and consistent user experience across different devices and screen sizes.

Technical Aspects

Diving into technicalities, the website consists of one CSS and four HTML documents alongside with Favicon package (more details about favicon in dissection below) and numerous .SVG, .PNG, and .JPG files to ensure an aesthetically pleasant experience. The requirements and specifications outlined by WebAss Specification page on canvas have been adhered to within the scope of various tags, classes, and id attributes.


The HTML structure of website consists of header and footer id attributes due to uniqueness of CSS styling properties targeting these elements. On the main section of website elements such as h1, p, ul, img, and svg get boxed via div tags and styled using class attributes. While considering technical aspects accessibility and robustness of the Hyper Text Markup Language and Cascading Style Sheets written play a pivotal role, that's why I adopted CSS variables declaring the font, colours, accent, and radius via root selector. For the final touch I practiced using rem and em units while declaring my properties to avoid any deformation of styles using hard units.

Structure

Talking about structure, the website benefits from a well-crafted structure that seamlessly adapts to different devices, ensuring an optimal browsing experience for all users. For desktop users, the structure features an intuitive navigation bar prominently displayed on the header { display: flex; }, providing easy access to key sections of the site. This structure allows a comprehensive view of the navigation options. Conversely, the mobile structure prioritizes simplicity and ease of use, featuring a navigation menu accessible via a menu button written solely using HTML input tag and CSS properties located on the right side of sticky header { display: inline-block; }. This mobile-friendly approach optimizes space and ensures effortless navigation on smaller screens, enhancing usability and accessibility.

To dissect the structure even further, it consists of three main elements each marked up with appropriate HTML tag:

  • Header: According to W3C HTML5 Specification document, the header element represents a group of introductory or navigational aids. The header element embedded in this website fulfills its purpose by nesting website's logo-box and navigation bar.

  • Main: The core of this website structure is the main element, which embeds several div elements with different class properties. These div elements are strategically positioned to organize content optimally, whether in rows or columns. This ensures a cohesive and visually appealing layout, facilitating readability and accessibility.

  • Footer: Completing the website structure is the footer, which serves as a final touchpoint for users. It includes a list of social media links, inviting users to connect beyond the website. Additionally, a copyright notice asserts ownership of the content and underscores my commitment to protecting intellectual property rights.

Favicon

Favicons are small but mighty icons that wield a significant impact on a website's identity and user experience. Typically sized at 16x16 and 32x32 pixels serving as the visual representation of a website in a browser's address bar or tabs. Their usage goes far more than branding; favicons enhance user navigation by providing quick visual cues, making it easier for users to identify and return to their favourite sites amidst a sea of tabs.


Favicons come in various formats, with ICO (Windows Icon Format) and PNG being the most common. Despite their size constraints, favicons offer huge benefits. They contribute to a polished user experience, lending a touch of professionalism and attention to detail. On my website I embedded these tiny magic boxes as a package (thanks to RealFaviconGenerator) considering compatibility for various browsers and devices. From XML document supporting IE browser (who uses that?!) to macOS touch bar!

Aesthetics

While designing my website I considered a minimalistic approach with extensive pickiness; from colour palette, to typography (font-weight and sizes). Choosing the font style I preferred the modern-looking font Mulish designed by Vernon Adams, Cyreal, and Jacques Le Bailly, to ensure the consistency I declared two backup fonts of Arial and sans-serif. Other considerations also played a huge role on my mindset while polishing the design such as consistency of elements (each div tag touched!), responsive design principles (testing on various device and browsers while shrinking webpage down and back up again) and images & graphics.


The visual aesthetics of my website rely heavily on images and graphics that I tried to embed in every corner. From photos taken by me on Home, Future, and Past pages to logos and illustrations designed via Canva (again me!) and last but certainly not least lovely svg designs by Katerina Limpitsouni available for commercial and non-commercial use on unDraw.

Setting against a background of pure white (#FFF), the canvas of website holds a minimalistic stand, inviting users to explore further. Shades of ocean green (#41BB87, #45A27A) add a refreshing splash, infusing a sense of clarity and calmness. Accessibility

While designing the website pages I endeavoured to ensure accessibility for all users. By prioritizing each of the accessibility guidelines and principles, outlined by WebAIM my website offers a seamless experience for all individuals no matter what (I hope). The accessibility principles implemented on the website:

  • Equivalent alternative text: Every image and non-text element on the website is accompanied by descriptive alternative text. This ensures that screen reader users can understand and navigate through the content effectively, enhancing their overall browsing experience.

  • Logical document structure: The website employs clear and hierarchical headings, lists, and other structural elements to provide meaning and organization to its content. This logical structure not only improves navigation for keyboard users, it also assists screen reader users in understanding the layout and flow of information.

  • Headers for data tables: Data tables on the website are embedded with row and column scopes, ensuring that screen reader users can navigate and comprehend tabular data effortlessly. This practice enhances accessibility for individuals who rely on assistive technologies to access information.

  • Links that make sense out of context: Each link on the website is crafted with descriptive anchor text that conveys its purpose and destination. By avoiding generic phrases like "click here" or "read more," I ensured that screen reader users can understand the context of each link, facilitating efficient navigation.

  • Avoiding reliance on color alone to convey meaning: The website does not rely solely on color to convey information or distinguish between elements. Instead, I used additional visual cues, such as icons or text labels, to ensure that content remains accessible to users with color vision deficiencies or those who disable browser stylesheets.

  • Designing to standards: By adhering to HTML and CSS standards, the website promotes accessibility and interoperability across different browsers and devices. Valid markup and clean code contribute to a more robust and flexible website architecture, enhancing accessibility for all users.
